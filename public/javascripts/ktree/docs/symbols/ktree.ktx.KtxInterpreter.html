<private> sendCommands_(parentName)
Send commands to the World. Two messages are sent: first, a sequence of interpreted KTX commands are dispatched (as function calls on the World object); second, a string of KML is sent to be rendered by the World. Note that this method does NOT check to see whether the World object's API is actually ready to receive commands. For that functionality, delaySendCommands_() should be used.
